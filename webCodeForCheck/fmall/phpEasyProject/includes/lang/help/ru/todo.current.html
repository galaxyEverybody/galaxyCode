Лист заданий (Что сделать)

Этот раздел состоит из двух частей - актуальных заданий и архива.
В обеих частях пользователи видят те проекты, к которым они включены в разделе управления проетами или разделе "Сотрудники".
Исключение составляют сотрудники со статусом ркуоводителя предприятия - они видят все задания, кроме помеченных другими как "личное".

Редактирование

Каждое задание имеет наименование и должно относится к конкретному проекту. Для лучшего контроля предусмотрены поля начала выполнения задания и опциональное поле для запланированного окончания задания. Из практики использование поля "Окончание" делает работу с заданиями много проще.

Каждое задание должно содержать приоритет. Кроме своего прямого назначения, возможно использование этого поля для специальных целей, используя кроме цифр также и буквы, например "Прод" - для раздела продаж или "П" для отдела разработчиков.

Список заданий не допускает смену создателя задания. Это означает, что раздел "Личное" при редактировании отключен для тех, кто это задание не создавал и стать таким образом "личным".

„Предупреждение о просрочивании“ имеет значение только в случае, если установлена запланированная дата окончания задания. Если срок для выполнения задания меньше установленного этой опцией, то "светофор" в колонке Статус списка для этого задания меняется на мигающий с предупреждением: "Внимание! Скоро будет просрочено". После окончания этого срока светофор меняет цвет на красный и тогда всё - туши свет, приехали!

Кроме того для лучшего обзора состояния заданий служит указание степени готовности задания, которое субъективно оценивается участником. В списке заданий степень готовности отображается зелёными балками.

Если задание выполненно, его следует отметить как выполненное нажатием на соответствующую опцию в списке заданий или при обработке в маске радактирования. При этом задание будет перечёркнуто горизонтальной линией для лучшего обзора и пущей привлекательности.

Поле "Комментарии" можно использовать как лист истории участников этого задания (с указанием времени и автора записи).

Любое задание в актуальном списке не может быть непосредственно удалено, а только перемещено в архив, откудо оно может быть безвозвратно (в зависимости от организации резервного сохранения данных) удалено.

При нажатии на зелёную иконку - крестик форма очищается для следующего задания. Щелчек по полю в списке задания загружает это задание в форму для обработки.

Лист заданий

Лист заданий разделен на страницы по 50 записей в каждой для лучшего обзора. Кроме того, для быстрого поиска служат фильтры "Поиск", "Прио.", типов заданий и конкретных проектов.
Щелчком по колонке описания в таблице производится сортировка данных по этой колонке.

Маркирование checkbox или щелчок по ячейке "Статус" перечеркивает задание (на быстрых серверах - быстро), делая его статус выполненным и обратно. при этом изменяется дата последнего изменения с запоминанием имени того, кто изменяет.

Щелчек мышью в ячейках "Ном.", "Прио.", "Задание" или "Проект" загружает задание в форму для редактирования.

Щелчек в поле готовности (Гот.) увеличивает степень готовности на 25%. После достижения 100% щетчик обнуляется и все повторяется сначала.

Значения светофора в поле "Статус":
  • Зелёный – Задание выполняется по плану (больше 0% выполнено)
  • Жёлтый – Врямя для выполнения пошло, но ещё ничего не сделано.
  • Желтый мигающий – Время для выполнения заканчивается (См. „Предупреждение о просрочивании“)
  • Красный – Время на выполнение задания окончилось.
  • Синий – Время для выполнения задания ещё не настало. Займитесь чем-нибудь другим.

Архив

Архив устроен аналогичным активным заданиям образом с той разницей, что здесь нельзя редактировать задания, зато их можно удалять. Для редактирования задания его нужно перенести из архива в "актуальные" щелчком по таблице со стрелкой.

Щелчёк на удаление стирает задание без возможности восстановления (в этой программе).
